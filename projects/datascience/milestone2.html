Project Goals

As the title suggests, the goal of this project is to use the public ovarian cancer dataset, a set of data that's well-regarded as good Top-down MS-based raw data, and use TopPIC, a tool created and maintained by our own lab, to do the general pipeline of workflow and analysis said raw data and lastly obtaining a list of proteoform identifications. Similar with all other available tools, TopPIC will have it's pros and cons as to the type of raw data it can identify the most, and so this project aims to explore what exactly those pros and cons are in comparsion to other major tools such as MSPathFinder or ProSight. Since ProSight requires a lisence, this project will focus on comparsions with MSPathFinder.

Project Background

This project deals with the minor field of Top-down MS-based proteomics, but its core is still about the performance of a certain analytical tool, hence very little biological knowledge is required to understand the meaningfulness of this project. Dispite that, I still need to explain a little more on this field and what exactly TopPIC does. The computer science/informatics side of modern-day proteomics is inherently about protein and proteoform (different variations of the same protein) identification. That is to say, given a sample of proteins, we need to figure out what this sample consists of, more specifically, what's the amino acid sequence of the proteins in the sample.

Top-down MS-based proteomics is one approach to this problem, that we take advantage of a machinery known as the mass spectrometry (MS), which is an analytical technique that is used to measure the mass-to-charge(m/z) ratio of ions. These m/z ratios would allow us to uniquely identify proteins/proteoforms using database search. But of course, in practice, the unique condition becomes a little rusty as noise and imprecision of the machinery comes into play while having to deal with huge computational complexity of searching through a database in terms of millions. And hence advance computational tools such as TopPIC were introduced to deal with these problems, that it would take the raw data generated by (MS), and return a set of protein/proteoform identifications.

Now there are essentially two things that defines a spectrum (unit of raw data), and impacts the database search. The MS1 feature, also known as precursor feature, and MS2 fragments, also known as peak list. Explaining why they act the way they are will dive into deep biology, so I will directly explain how they impact each other. The list of possbile precursor features needs to be first identified through a process known as feature detection, which is handled by TopFD, a part of the the TopPIC Suite. Then each spectra need to be manually assigned a precursor feature, which includes precursor mass, precursor charge, and adjusted precursor m/z. Identifiying the full list of precurosr and properly assign them to the spectra is crucial that they also play a part in identifying the list of peaks, which is what's searched against the database.

The process mentioned above is also being conducted through the MSPathFinder package as well, that they use ProMex for feature detection and MSPathFinderT for database search. One needs to note the difference in default input parameters, that it's very important to use the same set of parameters even though some might not be directly supported by the tools.

Project Datasets

This dataset can be found at MassIVE using the unique identifier MSV000080257 or accessed through this link. Do note that the datasets cannot be direclty downloaded through the browser, but through a FTP. And of course, this is 20 GB of public data for which is too much data to do a total analysis on, so we focus on a randomly chosen file labeled as CPTAC_Intact_rep1_15Jan15_Bane_C2-14-08-02RZ. And again, since this data is the raw data produced by the MS, we must run it through the Top-down search pipeline first to obtain data we need.

We will first run the raw dataset through a tool called MSConvert, which converts the raw data into a generally accepted format known as mzML, which is raw data parsed into xml with certain places compressed into hexadecimal incodings. We will then do feature detection using TopFD, a part of the TopPIC Suite, which would output us ms1.feature, ms2.feature, and ms2.align files. The first two files are tsv files under the hood which we will be looking at the first file. The last file is a txt file with certain structure that we standardized for database search. These three files will then be used as input to TopPIC, which performs the database search against a .fasta file downloaded from UniProt, and will output many different result files, and we are particularly interested in prsm_single.tsv.

Out of this process, there are two files that we will be looking at with close attetion. Namely ms1.feature, and prsm_single.tsv. As mentioned before, it's very important that we are assigning the correct precursor feature to each spectra, that a wrongly assigned precursor will likely to cause a difference in the peak list, but also possibly provide the wrong identification. Hence we must be looking at both the identification results and the list of features.

A similar process will happen using MSPathFinder as well, that we need to use the format of mzML to pass into PBFGen, a preprocessing tool as a part of the MSPathFinder package. This will return a output file with extension .pbf, which is custum file format acceptable by the rest of the package. It would then be thrown into ProMex with settings that sets the parameters the same as the default parameters of TopFD. Note that ProMex talks about minimum mass and minimum charge that's not part of the parameters of TopFD, but these two parameters are very unlikely to have an effect on the result as the internal default minimum charge is just 1, and the minimum mass varies based on the intensity of spectrum through a noise-signal ratio, which is a better filter than having a minimum mass directly being set. Now ProMex will output .ms1ft, which is the counterpart to ms1.feature files generated by TopFD. And lastly using both .ms1ft and .pbf as inputs, we can run MSPathFinderT against the same .fasta database, for which it would output _IcTarget.tsv, a counterpart to prsm_single.tsv. We see similar problems in MSPathFinder as well that it contains parameters not mentioned by TopPIC, yet they do not possess a real threat similar with reasons before.

Some of questions we will be answering through the project is: what type of spectra, or more specifically, what range of precursor are we more likely to identify and assign correctly using TopPIC in comparsion to MSPathFinder. Is there a preference in precursur mass, that perhaps higher precursur mass would more likely result in an protein identification? What type of spectra are TopPIC missing in comparsion with MSPathFinder and is there a trend as to the ones that we missed.

We need to first note that the 2 ms1 features are actually different tables. The ms1 features generated by TopPIC is the complete of all features and only of thoes features, whereas the features generated by MSPathFinder are the actual feature assignments for each ms2 scan. That the ms1 features provided by MSPathFinder is not a complete list, and there will also be overlaps about them. What this is likely to mean is that the feature list provided by MSPathFinder will likely to be useless as the information we can collect from that table is also inside the search result.

Furthermore, we can note that the search result reported by MSPathFinder does not have any modifications on them, which is really strange. This should either be a bug in their tool or that they don't expect unexpect modifications and therefore do not try to detect them. Hence I should generate the next round of search data with fixed modifications in mind and pass them in as parameters (This is no easy task as it will take hours for the tools to run even with multi-threading) (Or I should run it using our lab's server)

So at this point it seems that it's not very possible to correlate mass differences with other visible features in the search result, and that a model would be required.

One possible option for a model would be a classification predication model that uses all the existing features from the research result of TopPIC to predict the mass differences between precursor mass and proteoform mass, and since there exists a semi-clear distinction between 0 and 1.2, we can put class labels on them so things can be easy. Another option that I can explore is look at the scans missed by TopPIC but reported by MSPathFinder, and try to build a model of predication based on the feature informations in the missed scans and the feature informations in the reported scans. But since I only have the reported information from TopPIC, and the missed ones have to be obtained through a data format we created known as ms2.align. Such format cannot be placed into pandas dataframe/tidy format as each scan have a varibale number of ms2 fragments, each with intensity, charge, and m/z ratio.
